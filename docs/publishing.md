# Publishing Checklist

Use this checklist before making the repository public.

1. Confirm no source footage, music, rendered video, generated review frame, or copyrighted asset is committed.
2. Confirm `skills/game-bgm-video-maker/SKILL.md` describes the workflow without depending on one private project.
3. Confirm scripts run from a normal project workspace, not only from the original author environment.
4. Confirm `README.md` installation steps match the final repository URL.
5. Choose the intended license before public release.

Suggested first repository settings:

- visibility: public if you want broad reuse, private if you want to test with a few people first
- repository name: `game-bgm-video-maker-skill`
- default branch: `main`
- topics: `codex-skill`, `video-editing`, `bgm`, `avfoundation`, `youtube`
